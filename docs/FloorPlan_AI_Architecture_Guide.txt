SECTION 7 — VALIDATION RULES & ERROR HANDLING

7. Layout Validation Checklist
Every generated layout must pass ALL of these checks before SVG is rendered. Failures trigger an automatic correction pass (max 3 iterations). Remaining violations are surfaced to the user.

7.1 Hard Rules (must pass — generation fails if violated after 3 attempts)
Entry is on an exterior wall with a door opening to outside
No room has zero access from entry (every room reachable)
No bathroom or WC shares a wall with kitchen or dining room
Total rendered area is within 8% of requested gross area
No room bounding box overlaps any other room bounding box
Every door has 0.9m clearance in front of it (no furniture blocking)
No room is below its absolute minimum size (see Section 4)
Corridor width is never below 0.9m

7.2 Soft Rules (warnings shown to user, layout proceeds)
Living room does not face preferred direction (south if requested)
Bedroom has no exterior wall window
Corridor exceeds 3m when user asked to avoid long corridors
WC is not adjacent to entry (when 'near entry' was requested)
Open plan requested but partial wall remains
Room is below recommended size but above absolute minimum
Buanderie requested but only niche provided, not full room

7.3 Auto-Correction Priority Order
When violations are found, fix in this order:
1. Fix overlaps — shrink larger room first
2. Fix rooms below minimum — take space from corridor, then storage
3. Fix forbidden adjacencies — swap room positions
4. Fix missing access — add corridor segment
5. Fix area budget — proportionally resize all rooms
